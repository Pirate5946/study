### 权限修饰关键字
> public

> private

用于修饰类中的属性和方法，==只能在当前类中使用==，在别的类的方法中无法直接访问到private修饰的属性或方法

用 private 修饰构造方法，可以让类无法创建对象，只能访问静态成员
> static

- 用于修饰公共的成员变量，所有方法和对象共享
- 优先于对象存在，随着类的加载就存在了
- 可以直接用类名调用

**静态使用的注意事项**      

1、静态方法只能访问静态成员（属性或方法），没有使用对象的特有数据时可以用static修饰     
2、静态方法中不可以使用 this 或者 super 关键字        
3、主函数是静态的   

**main 函数的特殊之处**

1、格式是固定的     
public static void main(String[] args)   
2、被 jvm 识别和调用

**静态代码块**  

随着类的加载而执行，而且只执行一次

作用：用于给类进行初始化（区别于构造函数初始化，不需要对象），这种类里面基本都是静态方法和静态属性

### 静态变量和成员变量的区别    

1、两个变量的生命周期不同
- 成员变量随着对象的创建而存在，随着对象的被回收而释放。  
- 静态变量随着类的加载而存在，随着类的消失而消失   

2、调用方式不同 
- 成员变量只能被对象调用
- 静态变量可以被对象调用，还可以被类名调用

3、存储位置不同
- 成员变量数据存储在堆内存的对象中，所以也叫对象的特有数据
- 静态变量数据存储在方法区（的静态区），也叫对象的共享数据

> final 关键字

1. 可以修饰类、方法、变量
2. final修饰的类不可以被继承，防止类里面的方法被重写
3. final修饰的方法不可以被覆盖重写
4. final修饰的变量是一个常量，只能赋值一次，不允许修改，经常配合 static 使用

### this关键字
哪个对象调用this所属的方法，this就代表哪个对象的引用（栈内存中的变量）

this 代表它==所在函数的所属对象的引用==，对象内部方法的相互引用都是由对象来完成的，==一般可以省略 this==  
> this的用法

- 在方法体内向类的成员变量赋值时需要显式说明 this
- ==传入方法的对象或者属性==需要和方法所属对象区别开时
- 构造函数间相互调用，小心递归引用
    -  this语句(==this()==)只能定义在构造函数的第一行
    - （可以只对外暴露一个构造方法，通过this引用其他构造函数完成初始化）
